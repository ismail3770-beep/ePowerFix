১মূল সিদ্ধান্ত

ePowerFix প্ল্যাটফর্মের জন্য প্রফেশনাল ও স্কেলেবল আর্কিটেকচার নির্ধারণ করা হয়েছে। এই সিদ্ধান্তগুলো বাংলাদেশের প্রেক্ষাপটে ব্যবসার স্কেল, টিমের দক্ষতা, এবং ভবিষ্যৎ গ্রোথ বিবেচনা করে নেওয়া হয়েছে। মূল ফোকাস হলো এমন একটি সিস্টেম তৈরি করা যা শুরুতে কম খরচে চলবে, কিন্তু ইউজার বাড়ার সাথে সাথে সহজে স্কেল করা যাবে।

কোর সিদ্ধান্ত (৪টি)

নিচের চারটি কম্পোনেন্ট হলো সিস্টেমের মূল ভিত্তি। এগুলো ছাড়া সিস্টেম চলতে পারে না। প্রতিটি কম্পোনেন্ট নির্দিষ্ট দায়িত্ব পালন করে এবং একে অপরের সাথে সমন্বিতভাবে কাজ করে।

সিদ্ধান্তের পেছনের কারণ

এই স্ট্যাক বাংলাদেশের প্রেক্ষাপটে সেরা কারণ — TypeScript ভিত্তিক (টিমের একই স্কিল কাজে লাগে), বাংলাদেশি পেমেন্ট গেটওয়ে সাপোর্ট, ফ্রি টিয়ারে শুরু করা যায়, এবং ভবিষ্যতে সহজে স্কেল করা যায়। Flutter বা অন্য কোনো ভাষা ব্যবহার করলে বর্তমান Next.js কোড পুরো বর্বাদ হতো।

২কোর অ্যাপ্লিকেশন লেয়ার

কোর অ্যাপ্লিকেশন লেয়ার হলো সেই অংশ যা সরাসরি ইউজারের সাথে ইন্টারঅ্যাক্ট করে। এখানে তিনটি প্রধান অ্যাপ থাকবে — ওয়েব, মোবাইল, এবং API সার্ভার। প্রতিটি অ্যাপের নির্দিষ্ট দায়িত্ব থাকবে এবং এগুলো একে অপরের থেকে স্বাধীনভাবে কাজ করবে।

অ্যাপ স্ট্রাকচার

তিনটি অ্যাপ apps/ ফোল্ডারের অধীনে থাকবে। ওয়েব অ্যাপ শুধু UI দেখাবে, মোবাইল অ্যাপ Expo দিয়ে তৈরি হবে, এবং API সার্ভার Express দিয়ে তৈরি হবে। এই আর্কিটেকচারের সুবিধা হলো প্রতিটি অ্যাপ আলাদাভাবে ডিপ্লয় এবং স্কেল করা যায়।

শেয়ার্ড প্যাকেজ

packages/ ফোল্ডারে শেয়ার্ড কোড থাকবে যা তিনটি অ্যাপই ব্যবহার করবে। এর মধ্যে TypeScript টাইপ, API ক্লায়েন্ট, Zustand স্টোর, ইউটিলিটি ফাংশন, কনফিগ, এবং শেয়ার্ড UI কম্পোনেন্ট থাকবে। এর ফলে কোড ডুপ্লিকেশন এড়ানো যায় এবং এক জায়গায় পরিবর্তন করলে সব অ্যাপে রিফ্লেক্ট হয়।

প্রতিটি অ্যাপের দায়িত্ব

ওয়েব অ্যাপ শুধু ইউজার ইন্টারফেস দেখাবে — কোনো বিজনেস লজিক বা ডেটাবেস কোয়েরি এখানে থাকবে না। মোবাইল অ্যাপও একই নিয়মে চলবে। API সার্ভার সব বিজনেস লজিক, ডেটাবেস অপারেশন, পেমেন্ট প্রসেসিং, এবং অথেন্টিকেশন হ্যান্ডল করবে। এই সেপারেশনের ফলে সিস্টেম পরিষ্কার থাকে এবং মেইনটেইন করা সহজ হয়।

৩ডেটা ও স্টোরেজ লেয়ার

ডেটা লেয়ার হলো সিস্টেমের সবচেয়ে গুরুত্বপূর্ণ অংশ। এখানে শুধু মূল ডেটাবেস নয়, বরং ক্যাশিং, ইমেজ স্টোরেজ, এবং ফাইল স্টোরেজও থাকে। সঠির ডেটা লেয়ার ছাড়া সিস্টেম দ্রুত এবং নির্ভরযোগ্যভাবে চলতে পারে না।

PostgreSQL কেন?

PostgreSQL হলো বিশ্বের সবচেয়ে শক্তিশালী ওপেন-সোর্স রিলেশনাল ডেটাবেস। এটি ACID কমপ্লায়েন্ট, যার মানে ডেটা কখনো কারেপ্ট হবে না। JSON সাপোর্ট আছে, যাতে ফ্লেক্সিবল ডেটা স্টোর করা যায়। বাংলাদেশের সাইটগুলো থেকে সবচেয়ে দ্রুত অ্যাক্সেস পাওয়া যায় Singapore region থেকে, যেখানে Neon এবং Supabase উভয়ই সার্ভার রাখে। ফ্রি টিয়ারে ০.৫ GB স্টোরেজ পাওয়া যায়, যা ছোট ব্যবসার জন্য যথেষ্ট।

Prisma ORM কেন দরকার?

Prisma হলো একটি মডার্ন ORM যা TypeScript-এর সাথে পুরোপুরি কাজ করে। এটি টাইপ সেফটি দেয়, যার মানে কোডে টাইপ এরর ধরা পড়ে রানটাইমের আগেই। অটো-কমপ্লিট সাপোর্ট থাকে, তাই ডেভেলপার দ্রুত কোড লিখতে পারে। মাইগ্রেশন সিস্টেম থাকে, যাতে ডেটাবেস স্কিমা পরিবর্তন ট্র্যাক করা যায়। কোয়েরি অপটিমাইজেশন অটোমেটিক হয়, তাই ম্যানুয়ালি SQL অপটিমাইজ করার দরকার নেই।

Redis ক্যাশিং কেন?

Redis ব্যবহার করে যেসব ডেটা বারবার পড়া হয় কিন্তু কম বদলায়, সেগুলো ক্যাশে রাখা হয়। যেমন প্রোডাক্ট লিস্ট, ক্যাটাগরি, ইউজার সেশন। এর ফলে ডেটাবেসে কম লোড পড়ে এবং রেসপন্স টাইম ১০-১০০ গুণ দ্রুত হয়। Upstash সার্ভারলেস Redis প্রোভাইড করে যা ফ্রি টিয়ারে ১০,০০০ কমান্ড/দিন সাপোর্ট করে। রেট লিমিটিংও Redis দিয়ে করা হয়, যাতে API আবুস রোধ করা যায়।

Cloudinary কেন?

Cloudinary শুধু ইমেজ স্টোরেজ নয়, এটি একটি সম্পূর্ণ মিডিয়া ম্যানেজমেন্ট প্ল্যাটফর্ম। ইমেজ আপলোডের পর অটোমেটিকভাবে রিসাইজ, অপটিমাইজ, এবং বিভিন্ন ফরম্যাটে কনভার্ট করে। CDN সাপোর্ট থাকে, তাই ইমেজ দ্রুত লোড হয়। ফ্রি টিয়ারে ২৫ ক্রেডিট/মাস পাওয়া যায়, যা ছোট ব্যবসার জন্য যথেষ্ট। ওয়াটারমার্ক, ফেস ডিটেকশন, ব্যাকগ্রাউন্ড রিমুভাল ইত্যাদি ফিচারও আছে।

JSON Web Token (JWT) হলো স্টেটলেস অথেন্টিকেশনের স্ট্যান্ডার্ড। ইউজার লগইন করলে সার্ভার একটি JWT টোকেন দেয়, যা ক্লায়েন্ট পরবর্তী রিকোয়েস্টগুলোতে পাঠায়। এই টোকেনে ইউজারের আইডি এবং রোল এনকোডেড থাকে। টোকেনের মেয়াদ থাকে (সাধারণত ৭ দিন), তার পরে আবার লগইন করতে হয়। ওয়েবে কুকি দিয়ে টোকেন সেন্ড করা হয় (httpOnly, secure), আর মোবাইলে SecureStore-এ রাখা হয়।

পাসওয়ার্ড হ্যাশিং

পাসওয়ার্ড কখনো প্লেইন টেক্সটে স্টোর করা হয় না। bcrypt ব্যবহার করে পাসওয়ার্ড হ্যাশ করা হয়, যাতে এমনকি ডেটাবেস লিক হলেও পাসওয়ার্ড উদ্ধার করা যায় না। bcrypt স্লো হ্যাশিং অ্যালগরিদম ব্যবহার করে, যাতে brute force অ্যাটাক অসম্ভব হয়। সল্ট রাউন্ড ১০-১২ রাখা হয়, যা সিকিউরিটি এবং পারফরম্যান্সের ভারসাম্য রক্ষা করে।

রোল-বেসড অ্যাক্সেস কন্ট্রোল

তিনটি রোল থাকবে — ইউজার (সাধারণ কাস্টমার), স্টাফ (অর্ডার ম্যানেজ করতে পারে), এবং অ্যাডমিন (সব কিছু কন্ট্রোল করতে পারে)। প্রতিটি API এন্ডপয়েন্ট নির্দিষ্ট রোল চেক করবে। অ্যাডমিন প্যানেলে ঢোকার জন্য আলাদা লগইন পেজ থাকবে, যেখানে শুধু অ্যাডমিন রোলের ইউজার ঢুকতে পারবে। স্টাফ রোল সাব-অ্যাডমিন হিসেবে কাজ করবে — অর্ডার ম্যানেজ, কাস্টমার সাপোর্ট, কিন্তু সেটিংস পরিবর্তন করতে পারবে না।

অন্যান্য সিকিউরিটি ফিচার

Rate Limiting: প্রতি আইপি থেকে প্রতি মিনিটে সর্বোচ্চ ১০০ রিকোয়েস্ট অনুমোদিত। লগইন এন্ডপয়েন্টে আরও কড়া — প্রতি ১৫ মিনিটে ৫ বার। Redis-এ ট্র্যাক করা হয়।

CORS: শুধু নির্দিষ্ট ডোমেইন ও মোবাইল অ্যাপ থেকে রিকোয়েস্ট অনুমোদন। CSRF অ্যাটাক প্রতিরোধে সাহায্য করে।

Helmet: অটোমেটিক নিরাপদ HTTP হেডার সেট করে — XSS, clickjacking, MIME-type sniffing প্রতিরোধ করে।

Zod: সব ইনপুট ভ্যালিডেট করা হয়। SQL injection, XSS, এবং অন্যান্য ইনজেকশন অ্যাটাক প্রতিরোধ করে।

bKash ইন্টিগ্রেশন

bKash হলো বাংলাদেশের সবচেয়ে জনপ্রিয় মোবাইল ওয়ালেট। পেমেন্ট ফ্লো হলো — ইউজার চেকআউট করে, bKash-এ রিডাইরেক্ট হয়, পেমেন্ট কনফার্ম করে, তারপর আবার সাইটে ফিরে আসে। ওয়েবহুক দিয়ে পেমেন্ট স্ট্যাটাস ভেরিফাই করা হয়। বাংলাদেশে ৫ কোটি+ bKash ইউজার আছে, তাই এটি বাদ দেওয়া অসম্ভব। স্যান্ডবক্স টেস্টিং ফ্রি, লাইভ ক্রেডেনশিয়ালের জন্য ব্যবসার নিবন্ধন লাগে।

Nagad ইন্টিগ্রেশন

Nagad হলো বাংলাদেশ ডাক পরিদর্শন অধিদপ্তরের মোবাইল ওয়ালেট। bKash-এর মতোই কাজ করে। কম ট্রানজেকশন ফি (সর্বোচ্চ ১%), তাই ছোট অর্ডারের জন্য ভালো। মার্কেট শেয়ার বাড়ছে, বিশেষ করে গ্রামাঞ্চলে। পেমেন্ট ফ্লো bKash-এর মতোই — রিডাইরেক্ট, কনফার্ম, ওয়েবহুক ভেরিফিকেশন।

SSLCommerz কেন?

SSLCommerz হলো বাংলাদেশের সবচেয়ে বড় পেমেন্ট গেটওয়ে অ্যাগ্রিগেটর। একটি ইন্টিগ্রেশনে কার্ড (Visa, Mastercard, Amex), নেট ব্যাংকিং, এবং আরও অনেক মেথড সাপোর্ট করে। যারা bKash/Nagad ব্যবহার করেন না, তাদের জন্য এটি দরকার। কর্পোরেট ক্লায়েন্ট এবং B2B অর্ডারে কার্ড পেমেন্ট বেশি হয়।

Pathao ও RedX ডেলিভারি

ঢাকার বাইরে ডেলিভারির জন্য Pathao এবং RedX সবচেয়ে নির্ভরযোগ্য। পাথাও-এর API দিয়ে অর্ডার বুক করা, ট্র্যাকিং আইডি পাওয়া, স্ট্যাটাস চেক করা — সব অটোমেটেড। RedX-এরও একই রকম API। ভিতরে ঢাকায় নিজস্ব ডেলিভারি ম্যান থাকবে (সার্ভিস + প্রোডাক্ট ডেলিভারি), বাইরে কুরিয়ার। দাম — ঢাকার ভিতরে ৫০-৮০ টাকা, বাইরে ১০০-১৫০ টাকা।

Resend ইমেইল সার্ভিস

Resend হলো মডার্ন ইমেইল API যা ডেভেলপার-ফ্রেন্ডলি। ফ্রি টিয়ারে মাসে ৩০০০ ইমেইল, যা শুরুর জন্য যথেষ্ট। অর্ডার কনফার্মেশন, শিপিং আপডেট, পাসওয়ার্ড রিসেট, নিউজলেটার — সব ধরনের ইমেইল পাঠানো যায়। HTML টেমপ্লেট সাপোর্ট করে, তাই সুন্দর ইমেইল ডিজাইন করা যায়। ডেলিভারি রেট ৯৯%+, স্প্যামে যায় না।

Socket.io রিয়েল-টাইম

Socket.io হলো WebSocket-এর উপরে একটি লেয়ার যা রিয়েল-টাইম কমিউনিকেশন সহজ করে। ব্যবহার কেস — লাইভ অর্ডার ট্র্যাকিং (ডিলিভারি ম্যান কোথায় আছে), লাইভ চ্যাট সাপোর্ট, রিয়েল-টাইম স্টক আপডেট, এবং অ্যাডমিন ড্যাশবোর্ডে নতুন অর্ডার নোটিফিকেশন। Socket.io মিনি-সার্ভিস হিসেবে আলাদা পোর্টে চলবে (৩০০৩), Next.js এর সাথে ইন্টিগ্রেট হবে Caddy গেটওয়ে দিয়ে।

Firebase Cloud Messaging

Sentry হলো সবচেয়ে জনপ্রিয় এরর ট্র্যাকিং প্ল্যাটফর্ম। যখনই কোনো এরর হয় (ফ্রন্টএন্ড বা ব্যাকএন্ড), Sentry-তে অটোমেটিক রিপোর্ট যায়। প্রতিটি এররের সাথে স্ট্যাক ট্রেস, ইউজার কনটেক্সট, ব্রাউজার ইনফো থাকে। ডুপ্লিকেট এরর গ্রুপ করা হয়, তাই একই এরর হাজারবার হলেও একবার দেখায়। ইমেইল ও Slack এলার্ট আসে। ফ্রি টিয়ারে মাসে ৫০০০ এরর, যা ছোট প্রজেক্টের জন্য যথেষ্ট।

ePowerFix/ ├── apps/ │ ├── web/ ← Next.js (UI only) │ │ ├── src/ │ │ │ ├── app/ ← পেজ │ │ │ └── components/ ← ওয়েব কম্পোনেন্ট │ │ └── package.json │ │ │ ├── mobile/ ← Expo (mobile app) │ │ ├── app/ ← স্ক্রিন │ │ └── package.json │ │ │ └── api/ ← Express (backend) │ ├── src/ │ │ ├── routes/ ← API রুট │ │ ├── controllers/ ← বিজনেস লজিক │ │ ├── middleware/ ← auth, rateLimit, validate │ │ ├── services/ ← ডেটাবেস অপারেশন │ │ └── server.ts ← এন্ট্রি পয়েন্ট │ └── package.json │ ├── packages/ ← শেয়ার্ড কোড │ ├── types/ ← TypeScript টাইপ │ ├── api-client/ ← fetch wrapper │ ├── store/ ← Zustand স্টোর │ ├── utils/ ← ইউটিলিটি │ ├── config/ ← কনফিগ, এনভায়রনমেন্ট │ └── ui/ ← শেয়ার্ড UI (NativeWind) │ ├── prisma/ ← ডেটাবেস স্কিমা (shared) │ ├── schema.prisma │ └── migrations/ │ ├── docs/ ← ডকুমেন্টেশন ├── .github/workflows/ ← CI/CD ├── turbo.json └── package.json

ফোল্ডার ব্যাখ্যা

apps/web/ — Next.js ওয়েব অ্যাপ। শুধু UI কম্পোনেন্ট ও পেজ থাকবে। কোনো API রুট এখানে থাকবে না। Vercel-এ ডিপ্লয় হবে।

apps/mobile/ — Expo মোবাইল অ্যাপ। React Native স্ক্রিন থাকবে। EAS দিয়ে বিল্ড হবে। Play Store ও App Store-এ সাবমিট হবে।

apps/api/ — Express ব্যাকএন্ড। সব API রুট, বিজনেস লজিক, ডেটাবেস অপারেশন এখানে। Railway-এ ডিপ্লয় হবে।

packages/ — শেয়ার্ড কোড। তিনটি অ্যাপই এগুলো ব্যবহার করবে। টাইপ, API ক্লায়েন্ট, স্টোর, ইউটিলিটি — সব এক জায়গায়। পরিবর্তন করলে সব অ্যাপে রিফ্লেক্ট হয়।

prisma/ — ডেটাবেস স্কিমা ও মাইগ্রেশন। সব অ্যাপ এই স্কিমা ব্যবহার করে।
