## ১. মূল সিদ্ধান্ত

ePowerFix প্ল্যাটফর্মের জন্য প্রফেশনাল ও স্কেলেবল আর্কিটেকচার নির্ধারণ করা হয়েছে। এই সিদ্ধান্তগুলো বাংলাদেশের প্রেক্ষাপটে ব্যবসার স্কেল, টিমের দক্ষতা, এবং ভবিষ্যৎ গ্রোথ বিবেচনা করে নেওয়া হয়েছে। মূল ফোকাস হলো এমন একটি সিস্টেম তৈরি করা যা শুরুতে কম খরচে চলবে, কিন্তু ইউজার বাড়ার সাথে সাথে সহজে স্কেল করা যাবে।

### কোর সিদ্ধান্ত (৪টি)

| নং | কম্পোনেন্ট | টেকনোলজি | ভূমিকা |
|----|-----------|---------|--------|
| ১ | Monorepo | Turborepo | সব কোড একসাথে ম্যানেজ |
| ২ | Web | Next.js | শুধু UI/UX (পেজ, কম্পোনেন্ট) |
| ৩ | Backend | Express.js | API সার্ভার |
| ৪ | Database | PostgreSQL | ডেটা সংরক্ষণ |

এই ৪টি হলো মূল ভিত্তি। কিন্তু একটি প্রোডাকশন-রেডি সিস্টেমের জন্য এগুলো ছাড়াও আরও কিছু কম্পোনেন্ট প্রয়োজন, যা নিচের সেকশনগুলোতে বিস্তারিত আলোচনা করা হয়েছে। প্রতিটি কম্পোনেন্ট নির্দিষ্ট সমস্যা সমাধানের জন্য যোগ করা হয়েছে এবং এগুলো বাদ দিলে সিস্টেম প্রোডাকশনে টিকবে না।

## ২. কোর অ্যাপ্লিকেশন লেয়ার

কোর অ্যাপ্লিকেশন লেয়ার হলো সেই অংশ যা সরাসরি ইউজারের সাথে ইন্টারঅ্যাক্ট করে। এখানে তিনটি প্রধান অ্যাপ থাকবে — ওয়েব, মোবাইল, এবং API সার্ভার। প্রতিটি অ্যাপের নির্দিষ্ট দায়িত্ব থাকবে এবং এগুলো একে অপরের থেকে স্বাধীনভাবে কাজ করবে।

### অ্যাপ স্ট্রাকচার

তিনটি অ্যাপ `apps/` ফোল্ডারের অধীনে থাকবে। ওয়েব অ্যাপ শুধু UI দেখাবে, মোবাইল অ্যাপ Expo দিয়ে তৈরি হবে, এবং API সার্ভার Express দিয়ে তৈরি হবে। এই আর্কিটেকচারের সুবিধা হলো প্রতিটি অ্যাপ আলাদাভাবে ডিপ্লয় এবং স্কেল করা যায়। ওয়েব অ্যাপ Vercel-এ হোস্ট হবে, API সার্ভার Railway বা Render-এ হোস্ট হবে, এবং মোবাইল অ্যাপ EAS দিয়ে বিল্ড হবে।

### শেয়ার্ড প্যাকেজ

`packages/` ফোল্ডারে শেয়ার্ড কোড থাকবে যা তিনটি অ্যাপই ব্যবহার করবে। এর মধ্যে TypeScript টাইপ, API ক্লায়েন্ট, Zustand স্টোর, ইউটিলিটি ফাংশন, কনফিগ, এবং শেয়ার্ড UI কম্পোনেন্ট থাকবে। এর ফলে কোড ডুপ্লিকেশন এড়ানো যায় এবং এক জায়গায় পরিবর্তন করলে সব অ্যাপে রিফ্লেক্ট হয়।

### প্রতিটি অ্যাপের দায়িত্ব

ওয়েব অ্যাপ শুধু ইউজার ইন্টারফেস দেখাবে — কোনো বিজনেস লজিক বা ডেটাবেস কোয়েরি এখানে থাকবে না। মোবাইল অ্যাপও একই নিয়মে চলবে। API সার্ভার সব বিজনেস লজিক, ডেটাবেস অপারেশন, পেমেন্ট প্রসেসিং, এবং অথেন্টিকেশন হ্যান্ডল করবে। এই সেপারেশনের ফলে সিস্টেম পরিষ্কার থাকে এবং মেইনটেইন করা সহজ হয়।

## ৩. ডেটা ও স্টোরেজ লেয়ার

ডেটা লেয়ার হলো সিস্টেমের সবচেয়ে গুরুত্বপূর্ণ অংশ। এখানে শুধু মূল ডেটাবেস নয়, বরং ক্যাশিং, ইমেজ স্টোরেজ, এবং ফাইল স্টোরেজও থাকে। সঠির ডেটা লেয়ার ছাড়া সিস্টেম দ্রুত এবং নির্ভরযোগ্যভাবে চলতে পারে না।

| কম্পোনেন্ট | টেকনোলজি | কেন দরকার |
|-----------|---------|----------|
| Database | PostgreSQL | মূল ডেটা (প্রোডাক্ট, অর্ডার, ইউজার) |
| ORM | Prisma | ডেটাবেস কোয়েরি সহজ করতে |
| Cache | Redis (Upstash) | দ্রুত ডেটা অ্যাক্সেস, রেট লিমিট |
| Image Storage | Cloudinary | প্রোডাক্ট ইমেজ স্টোর ও অপ্টিমাইজ |
| File Storage | AWS S3 / Cloudinary | PDF, ডকুমেন্ট |

### PostgreSQL কেন?

PostgreSQL হলো বিশ্বের সবচেয়ে শক্তিশালী ওপেন-সোর্স রিলেশনাল ডেটাবেস। এটি ACID কমপ্লায়েন্ট, যার মানে ডেটা কখনো কারেপ্ট হবে না। JSON সাপোর্ট আছে, যাতে ফ্লেক্সিবল ডেটা স্টোর করা যায়। বাংলাদেশের সাইটগুলো থেকে সবচেয়ে দ্রুত অ্যাক্সেস পাওয়া যায় Singapore region থেকে, যেখানে Neon এবং Supabase উভয়ই সার্ভার রাখে। ফ্রি টিয়ারে ০.৫ GB স্টোরেজ পাওয়া যায়, যা ছোট ব্যবসার জন্য যথেষ্ট।

### Prisma কেন দরকার?

Prisma হলো একটি মডার্ন ORM যা TypeScript-এর সাথে পুরোপুরি কাজ করে। এটি টাইপ সেফটি দেয়, যার মানে কোডে টাইপ এরর ধরা পড়ে রানটাইমের আগেই। অটো-কমপ্লিট সাপোর্ট থাকে, তাই ডেভেলপার দ্রুত কোড লিখতে পারে। মাইগ্রেশন সিস্টেম থাকে, যাতে ডেটাবেস স্কিমা পরিবর্তন ট্র্যাক করা যায়। কোয়েরি অপটিমাইজেশন অটোমেটিক হয়, তাই ম্যানুয়ালি SQL অপটিমাইজ করার দরকার নেই।

### Redis ক্যাশিং কেন?

Redis ব্যবহার করে যেসব ডেটা বারবার পড়া হয় কিন্তু কম বদলায়, সেগুলো ক্যাশে রাখা হয়। যেমন প্রোডাক্ট লিস্ট, ক্যাটাগরি, ইউজার সেশন। এর ফলে ডেটাবেসে কম লোড পড়ে এবং রেসপন্স টাইম ১০-১০০ গুণ দ্রুত হয়। Upstash সার্ভারলেস Redis প্রোভাইড করে যা ফ্রি টিয়ারে ১০,০০০ কমান্ড/দিন সাপোর্ট করে। রেট লিমিটিংও Redis দিয়ে করা হয়, যাতে API আবুস রোধ করা যায়।

### Cloudinary কেন?

Cloudinary শুধু ইমেজ স্টোরেজ নয়, এটি একটি সম্পূর্ণ মিডিয়া ম্যানেজমেন্ট প্ল্যাটফর্ম। ইমেজ আপলোডের পর অটোমেটিকভাবে রিসাইজ, অপটিমাইজ, এবং বিভিন্ন ফরম্যাটে কনভার্ট করে। CDN সাপোর্ট থাকে, তাই ইমেজ দ্রুত লোড হয়। ফ্রি টিয়ারে ২৫ ক্রেডিট/মাস পাওয়া যায়, যা ছোট ব্যবসার জন্য যথেষ্ট। ওয়াটারমার্ক, ফেস ডিটেকশন, ব্যাকগ্রাউন্ড রিমুভাল ইত্যাদি ফিচারও আছে।

## ৪. অথেন্টিকেশন ও সিকিউরিটি

সিকিউরিটি হলো যেকোনো অনলাইন প্ল্যাটফর্মের সবচেয়ে গুরুত্বপূর্ণ দিক। একটি ই-কমার্স প্ল্যাটফর্মে ইউজারের ব্যক্তিগত তথ্য, পেমেন্ট ডিটেইলস, এবং অর্ডার হিস্ট্রি থাকে — তাই সিকিউরিটি কম্প্রোমাইজ হলে বড় ধরনের ক্ষতি হতে পারে।

| কম্পোনেন্ট | টেকনোলজি | কেন দরকার |
|-----------|---------|----------|
| Auth | JWT + bcrypt | লগইন, রেজিস্টার, পাসওয়ার্ড হ্যাশ |
| Admin Auth | Role-based (admin/staff/user) | অ্যাডমিন প্যানেল সিকিউর করতে |
| Rate Limiting | Express middleware | API আবুস রোধ |
| CORS | cors middleware | মোবাইল থেকে API কল |
| Helmet | helmet middleware | HTTP হেডার সিকিউর |
| Input Validation | Zod | রিকোয়েস্ট ভ্যালিডেশন |

### JWT অথেন্টিকেশন

JSON Web Token (JWT) হলো স্টেটলেস অথেন্টিকেশনের স্ট্যান্ডার্ড। ইউজার লগইন করলে সার্ভার একটি JWT টোকেন দেয়, যা ক্লায়েন্ট পরবর্তী রিকোয়েস্টগুলোতে পাঠায়। এই টোকেনে ইউজারের আইডি এবং রোল এনকোডেড থাকে। টোকেনের মেয়াদ থাকে (সাধারণত ৭ দিন), তার পরে আবার লগইন করতে হয়। ওয়েবে কুকি দিয়ে টোকেন সেন্ড করা হয় (httpOnly, secure), আর মোবাইলে SecureStore-এ রাখা হয়।

### পাসওয়ার্ড হ্যাশিং

পাসওয়ার্ড কখনো প্লেইন টেক্সটে স্টোর করা হয় না। bcrypt ব্যবহার করে পাসওয়ার্ড হ্যাশ করা হয়, যাতে এমনকি ডেটাবেস লিক হলেও পাসওয়ার্ড উদ্ধার করা যায় না। bcrypt স্লো হ্যাশিং অ্যালগরিদম ব্যবহার করে, যাতে brute force অ্যাটাক অসম্ভব হয়। সল্ট রাউন্ড ১০-১২ রাখা হয়, যা সিকিউরিটি এবং পারফরম্যান্সের ভারসাম্য রক্ষা করে।

### রোল-বেসড অ্যাক্সেস কন্ট্রোল

তিনটি রোল থাকবে — ইউজার (সাধারণ কাস্টমার), স্টাফ (অর্ডার ম্যানেজ করতে পারে), এবং অ্যাডমিন (সব কিছু কন্ট্রোল করতে পারে)। প্রতিটি API এন্ডপয়েন্ট নির্দিষ্ট রোল চেক করবে। অ্যাডমিন প্যানেলে ঢোকার জন্য আলাদা লগইন পেজ থাকবে, যেখানে শুধু অ্যাডমিন রোলের ইউজার ঢুকতে পারবে। স্টাফ রোল সাব-অ্যাডমিন হিসেবে কাজ করবে — অর্ডার ম্যানেজ, কাস্টমার সাপোর্ট, কিন্তু সেটিংস পরিবর্তন করতে পারবে না।

### রেট লিমিটিং

রেট লিমিটিং ছাড়া একজন অ্যাটাকার API-তে হাজার হাজার রিকোয়েস্ট পাঠিয়ে সার্ভার ডাউন করতে পারে। Express middleware দিয়ে প্রতি আইপি থেকে প্রতি মিনিটে সর্বোচ্চ ১০০ রিকোয়েস্ট অনুমোদিত। লগইন এন্ডপয়েন্টে আরও কড়া — প্রতি ১৫ মিনিটে ৫ বার। রেট লিমিট Redis-এ ট্র্যাক করা হয়, তাই সার্ভার রিস্টার্ট হলেও লিমিট রিসেট হয় না।

### CORS কনফিগারেশন

Cross-Origin Resource Sharing (CORS) কনফিগার না করলে মোবাইল অ্যাপ থেকে API কল ব্লক হবে। শুধু নির্দিষ্ট ডোমেইন (ওয়েব) এবং মোবাইল অ্যাপ থেকে রিকোয়েস্ট অনুমোদন করা হবে। অন্য সব অরিজিন ব্লক। এটি CSRF অ্যাটাক প্রতিরোধে সাহায্য করে।

### Helmet মিডলওয়্যার

Helmet অটোমেটিকভাবে নিরাপদ HTTP হেডার সেট করে — X-Content-Type-Options, X-Frame-Options, Strict-Transport-Security ইত্যাদি। এগুলো XSS, clickjacking, এবং MIME-type sniffing অ্যাটাক প্রতিরোধ করে। এক লাইন কোডে সিকিউরিটি হেডার সেট হয়ে যায়।

### Zod ভ্যালিডেশন

ক্লায়েন্ট থেকে আসা সব ইনপুট ভ্যালিডেট করা হয় Zod স্কিমা দিয়ে। যেমন রেজিস্টার রিকোয়েস্টে ইমেইল ফরম্যাট, পাসওয়ার্ড লেংথ, ফোন নম্বর ফরম্যাট চেক করা হয়। ভ্যালিডেশন ব্যর্থ হলে ৪০০ এরর রিটার্ন করা হয় স্পেসিফিক মেসেজ সহ। এটি SQL injection, XSS, এবং অন্যান্য ইনজেকশন অ্যাটাক প্রতিরোধ করে।

## ৫. পেমেন্ট ও ইন্টিগ্রেশন

পেমেন্ট সিস্টেম হলো ই-কমার্স প্ল্যাটফর্মের হৃদপিণ্ড। বাংলাদেশে মোবাইল ওয়ালেট (bKash, Nagad) সবচেয়ে জনপ্রিয়, তাই এগুলো অবশ্যই সাপোর্ট করতে হবে। পাশাপাশি কার্ড পেমেন্টের জন্য SSLCommerz থাকবে। ডেলিভারির জন্য Pathao এবং RedX ইন্টিগ্রেট করা হবে।

| কম্পোনেন্ট | কেন দরকার |
|-----------|----------|
| bKash API | বাংলাদেশি পেমেন্ট |
| Nagad API | বাংলাদেশি পেমেন্ট |
| SSLCommerz | কার্ড/ব্যাংক পেমেন্ট |
| Pathao/RedX API | ডেলিভারি (ঢাকার বাইরে) |
| Email (Resend/SendGrid) | অর্ডার কনফার্মেশন, নোটিফিকেশন |
| SMS (SSL/ReveSMS) | OTP, অর্ডার আপডেট |

### bKash ইন্টিগ্রেশন

bKash হলো বাংলাদেশের সবচেয়ে জনপ্রিয় মোবাইল ওয়ালেট। পেমেন্ট ফ্লো হলো — ইউজার চেকআউট করে, bKash-এ রিডাইরেক্ট হয়, পেমেন্ট কনফার্ম করে, তারপর আবার সাইটে ফিরে আসে। ওয়েবহুক দিয়ে পেমেন্ট স্ট্যাটাস ভেরিফাই করা হয়। বাংলাদেশে ৫ কোটি+ bKash ইউজার আছে, তাই এটি বাদ দেওয়া অসম্ভব। স্যান্ডবক্স টেস্টিং ফ্রি, লাইভ ক্রেডেনশিয়ালের জন্য ব্যবসার নিবন্ধন লাগে।

### Nagad ইন্টিগ্রেশন

Nagad হলো বাংলাদেশ ডাক পরিদর্শন অধিদপ্তরের মোবাইল ওয়ালেট। bKash-এর মতোই কাজ করে। কম ট্রানজেকশন ফি (সর্বোচ্চ ১%), তাই ছোট অর্ডারের জন্য ভালো। মার্কেট শেয়ার বাড়ছে, বিশেষ করে গ্রামাঞ্চলে। পেমেন্ট ফ্লো bKash-এর মতোই — রিডাইরেক্ট, কনফার্ম, ওয়েবহুক ভেরিফিকেশন।

### SSLCommerz কেন?

SSLCommerz হলো বাংলাদেশের সবচেয়ে বড় পেমেন্ট গেটওয়ে অ্যাগ্রিগেটর। একটি ইন্টিগ্রেশনে কার্ড (Visa, Mastercard, Amex), নেট ব্যাংকিং, এবং আরও অনেক মেথড সাপোর্ট করে। যারা bKash/Nagad ব্যবহার করেন না, তাদের জন্য এটি দরকার। কর্পোরেট ক্লায়েন্ট এবং B2B অর্ডারে কার্ড পেমেন্ট বেশি হয়। ফ্রি স্যান্ডবক্স, লাইভ ক্রেডেনশিয়ালের জন্য ব্যবসার নিবন্ধন এবং ব্যাংক গ্যারান্টি লাগে।

### Pathao ও RedX ডেলিভারি

ঢাকার বাইরে ডেলিভারির জন্য Pathao এবং RedX সবচেয়ে নির্ভরযোগ্য। পাথাও-এর API দিয়ে অর্ডার বুক করা, ট্র্যাকিং আইডি পাওয়া, স্ট্যাটাস চেক করা — সব অটোমেটেড। RedX-এরও একই রকম API। ভিতরে ঢাকায় নিজস্ব ডেলিভারি ম্যান থাকবে (সার্ভিস + প্রোডাক্ট ডেলিভারি), বাইরে কুরিয়ার। দাম — ঢাকার ভিতরে ৫০-৮০ টাকা, বাইরে ১০০-১৫০ টাকা।

### Resend ইমেইল সার্ভিস

Resend হলো মডার্ন ইমেইল API যা ডেভেলপার-ফ্রেন্ডলি। ফ্রি টিয়ারে মাসে ৩০০০ ইমেইল, যা শুরুর জন্য যথেষ্ট। অর্ডার কনফার্মেশন, শিপিং আপডেট, পাসওয়ার্ড রিসেট, নিউজলেটার — সব ধরনের ইমেইল পাঠানো যায়। HTML টেমপ্লেট সাপোর্ট করে, তাই সুন্দর ইমেইল ডিজাইন করা যায়। ডেলিভারি রেট ৯৯%+, স্প্যামে যায় না।

### SMS সার্ভিস

OTP ভেরিফিকেশন এবং অর্ডার আপডেটের জন্য SMS দরকার। SSL Wireless বা Reve SMS বাংলাদেশে জনপ্রিয়। প্রতি SMS ০.৩০-০.৫০ টাকা। ফোন নম্বর ভেরিফিকেশন, অর্ডার স্ট্যাটাস, ডেলিভারি কনফার্মেশন — এসব ক্ষেত্রে SMS পাঠানো হয়। বাল্ক SMS সার্ভিস দিয়ে প্রমোশনাল মেসেজও পাঠানো যায়।

## ৬. রিয়েল-টাইম ও নোটিফিকেশন

রিয়েল-টাইম ফিচার মডার্ন ই-কমার্স প্ল্যাটফর্মের জন্য গুরুত্বপূর্ণ। কাস্টমার চায় অর্ডার করার সাথে সাথে স্ট্যাটাস আপডেট দেখতে, চ্যাট সাপোর্টে তাৎক্ষণিক রিপ্লাই পেতে। পুশ নোটিফিকেশন ইউজারকে আবার অ্যাপে ফিরিয়ে আনে।

| কম্পোনেন্ট | টেকনোলজি | কেন দরকার |
|-----------|---------|----------|
| WebSocket | Socket.io | লাইভ চ্যাট, অর্ডার ট্র্যাকিং |
| Push Notification | Firebase Cloud Messaging (FCM) | মোবাইল নোটিফিকেশন |
| Web Push | Web Push API | ওয়েব PWA নোটিফিকেশন |

### Socket.io রিয়েল-টাইম

Socket.io হলো WebSocket-এর উপরে একটি লেয়ার যা রিয়েল-টাইম কমিউনিকেশন সহজ করে। ব্যবহার কেস — লাইভ অর্ডার ট্র্যাকিং (ডিলিভারি ম্যান কোথায় আছে), লাইভ চ্যাট সাপোর্ট, রিয়েল-টাইম স্টক আপডেট, এবং অ্যাডমিন ড্যাশবোর্ডে নতুন অর্ডার নোটিফিকেশন। Socket.io মিনি-সার্ভিস হিসেবে আলাদা পোর্টে চলবে (৩০০৩), Next.js এর সাথে ইন্টিগ্রেট হবে Caddy গেটওয়ে দিয়ে।

### Firebase Cloud Messaging

FCM হলো গুগলের ফ্রি পুশ নোটিফিকেশন সার্ভিস। মোবাইল অ্যাপে পুশ নোটিফিকেশন পাঠাতে এটি স্ট্যান্ডার্ড। কাস্টমার অ্যাপ ইনস্টল করলে একটি ডিভাইস টোকেন পাওয়া যায়, সেটি ডেটাবেসে সেভ করা হয়। অর্ডার আপডেট, ডিসকাউন্ট অফার, নতুন প্রোডাক্ট — এসব নোটিফিকেশন পাঠানো যায়। ফ্রি আনলিমিটেড, শুধু মেসেজ পেলোড ৪KB এর নিচে রাখতে হয়।

### Web Push API

PWA ইউজারদের জন্য Web Push API ব্যবহার করা হয়। ব্রাউজার থেকে নোটিফিকেশন পারমিশন নিয়ে, সার্ভিস ওয়ার্কার দিয়ে নোটিফিকেশন দেখানো হয়। ইউজার অ্যাপ ইনস্টল না করলেও নোটিফিকেশন পায়। এটি FCM এর সাথে ইন্টিগ্রেট করা যায়, তাই একই ব্যাকএন্ড থেকে মোবাইল ও ওয়েব দুটোতেই নোটিফিকেশন পাঠানো যায়।

## ৭. অবজারভেবিলিটি ও মনিটরিং

প্রোডাকশনে সিস্টেম চালু হওয়ার পর সবকিছু ঠিক চলছে কিনা মনিটর করা জরুরি। কোনো এরর হলে দ্রুত ধরতে হবে, পারফরম্যান্স সমস্যা থাকলে আগে থেকে জানতে হবে। এই মনিটরিং ছাড়া সিস্টেম "কাজ করছে" ভেবে বসে থাকলে বড় সমস্যা হতে পারে।

| কম্পোনেন্ট | টেকনোলজি | কেন দরকার |
|-----------|---------|----------|
| Error Tracking | Sentry | প্রোডাকশন এরর ট্র্যাক |
| Logging | Pino / Winston | স্ট্রাকচার্ড লগ |
| Analytics | PostHog / Google Analytics | ইউজার বিহেভিয়ার |
| Uptime Monitoring | BetterStack / UptimeRobot | সাইট ডাউন হলে এলার্ট |
| APM | (বড় হলে) New Relic | পারফরম্যান্স মনিটরিং |

### Sentry এরর ট্র্যাকিং

Sentry হলো সবচেয়ে জনপ্রিয় এরর ট্র্যাকিং প্ল্যাটফর্ম। যখনই কোনো এরর হয় (ফ্রন্টএন্ড বা ব্যাকএন্ড), Sentry-তে অটোমেটিক রিপোর্ট যায়। প্রতিটি এররের সাথে স্ট্যাক ট্রেস, ইউজার কনটেক্সট, ব্রাউজার ইনফো থাকে। ডুপ্লিকেট এরর গ্রুপ করা হয়, তাই একই এরর হাজারবার হলেও একবার দেখায়। ইমেইল ও Slack এলার্ট আসে। ফ্রি টিয়ারে মাসে ৫০০০ এরর, যা ছোট প্রজেক্টের জন্য যথেষ্ট।

### Pino স্ট্রাকচার্ড লগিং

Pino হলো সবচেয়ে দ্রুত Node.js লগার। লগ JSON ফরম্যাটে থাকে, তাই সহজে সার্চ ও ফিল্টার করা যায়। প্রতিটি রিকোয়েস্টে রিকোয়েস্ট আইডি, ইউজার আইডি, রেসপন্স টাইম, স্ট্যাটাস কোড লগ করা হয়। প্রোডাকশনে সমস্যা ডিবাগ করার সময় এই লগ অমূল্য। লগ ফাইলের সাইজ কম, পারফরম্যান্সে প্রভাব নেই।

### PostHog অ্যানালিটিক্স

PostHog হলো ওপেন-সোর্স প্রোডাক্ট অ্যানালিটিক্স। গুগল অ্যানালিটিক্সের বিকল্প কিন্তু বেশি প্রাইভেসি-ফ্রেন্ডলি। ইউজার বিহেভিয়ার ট্র্যাক, ফানেল অ্যানালাইসিস, কনভার্সন ট্র্যাকিং, A/B টেস্টিং — সব ফিচার আছে। সেশন রিপ্লে ফিচার দিয়ে ইউজার কীভাবে সাইট ব্যবহার করে সেটা ভিডিওর মতো দেখা যায়। সেলফ-হোস্টেড ভার্সন ফ্রি, ক্লাউড ভার্সনে ফ্রি টিয়ার আছে।

### UptimeRobot মনিটরিং

UptimeRobot প্রতি ৫ মিনিটে সাইট চেক করে। ডাউন হলে ইমেইল, SMS, বা Slack এলার্ট পাঠায়। ফ্রি টিয়ারে ৫০ মনিটর, ৫ মিনিট ইন্টারভাল। পাবলিক স্ট্যাটাস পেজ তৈরি করা যায় যা কাস্টমার দেখতে পারে। এটি ছাড়া সাইট ডাউন হলে হয়তো ঘণ্টার পর ঘণ্টা পর জানতে পারবেন।

## ৮. ইনফ্রাস্ট্রাকচার ও ডিপ্লয়মেন্ট

ইনফ্রাস্ট্রাকচার হলো সেই ভিত্তি যার উপর পুরো সিস্টেম দাঁড়িয়ে থাকে। সঠিক হোস্টিং প্রোভাইডার বাছাই করা জরুরি — এটি পারফরম্যান্স, নির্ভরযোগ্যতা, এবং খরচের উপর সরাসরি প্রভাব ফেলে। বাংলাদেশি ইউজারদের জন্য Singapore region সবচেয়ে ভালো, কারণ লেটেন্সি কম।

| কম্পোনেন্ট | টেকনোলজি | কেন দরকার |
|-----------|---------|----------|
| Web Hosting | Vercel | Next.js এর জন্য সেরা |
| API Hosting | Railway / Render / DigitalOcean | Express সার্ভার |
| Database Hosting | Neon / Supabase | ম্যানেজড PostgreSQL |
| Redis Hosting | Upstash | সার্ভারলেস Redis |
| CDN | Vercel Edge / Cloudflare | দ্রুত কন্টেন্ট ডেলিভারি |
| SSL | Vercel/Let's Encrypt (ফ্রি) | HTTPS |
| Domain | .com.bd | বাংলাদেশি ট্রাস্ট |

### Vercel ওয়েব হোস্টিং

Vercel হলো Next.js এর নির্মাতা প্রতিষ্ঠান, তাই Next.js অ্যাপের জন্য সবচেয়ে ভালো হোস্টিং। অটো ডিপ্লয় — git push করলেই সাইট আপডেট হয়ে যায়। গ্লোবাল CDN, তাই বাংলাদেশ থেকেও দ্রুত লোড হয়। এজ ফাংশন সাপোর্ট, যাতে ক্যাশিং এবং রিডাইরেক্ট এজে চলে। ফ্রি টিয়ারে পর্যাপ্ত ব্যান্ডউইডথ, ছোট ব্যবসার জন্য যথেষ্ট। কাস্টম ডোমেইন, SSL, অটো রিনিউয়াল — সব ফ্রি।

### Railway API হোস্টিং

Railway হলো মডার্ন API হোস্টিং প্ল্যাটফর্ম। Express সার্ভার ডিপ্লয় করা সহজ — git push করলেই বিল্ড ও ডিপ্লয় হয়। পরিবেশ ভেরিয়েবল ম্যানেজমেন্ট, লগ ভিউয়ার, মেট্রিক্স — সব বিল্ট-ইন। ফ্রি টিয়ারে মাসে ৫০০ ঘণ্টা, যা শুরুর জন্য যথেষ্ট। Render এবং DigitalOcean ও Alternatives। DigitalOcean আরও কন্ট্রোল দেয় কিন্তু ম্যানেজ করতে হয়।

### Neon ডেটাবেস হোস্টিং

Neon হলো সার্ভারলেস PostgreSQL। স্কেল-টু-জিরো ফিচার — কোনো রিকোয়েস্ট না থাকলে ডেটাবেস স্লিপ মোডে চলে যায়, খরচ বাঁচে। ব্রাঞ্চিং — git ব্রাঞ্চের মতো ডেটাবেস ব্রাঞ্চ করা যায়, টেস্টিং সহজ। ফ্রি টিয়ারে ০.৫ GB স্টোরেজ, যা ৫০,০০০ প্রোডাক্ট পর্যন্ত সাপোর্ট করে। অটো ব্যাকআপ, পয়েন্ট-ইন-টাইম রিকভারি — সব ফিচার আছে। Supabase ও Alternatives কিন্তু Neon বেশি সিম্পল।

### Upstash Redis

Upstash হলো সার্ভারলেস Redis। পে-পার-রিকোয়েস্ট মডেল, তাই ব্যবহার না করলে খরচ নেই। ফ্রি টিয়ারে দৈনিক ১০,০০০ কমান্ড, যা ছোট ব্যবসার জন্য যথেষ্ট। গ্লোবাল এন্ডপয়েন্ট, তাই দ্রুত অ্যাক্সেস। রেট লিমিটিং, সেশন স্টোরেজ, ক্যাশিং — সব কাজে ব্যবহার করা যায়।

### Cloudflare CDN

Cloudflare হলো বিশ্বের সবচেয়ে বড় CDN প্রোভাইডার। স্ট্যাটিক অ্যাসেট (CSS, JS, ইমেজ) গ্লোবালি ক্যাশ করে, তাই বাংলাদেশ থেকেও দ্রুত লোড হয়। DDoS প্রোটেকশন ফ্রি। DNS ম্যানেজমেন্ট সহজ। ফ্রি প্ল্যানে আনলিমিটেড ব্যান্ডউইডথ। Vercel এর নিজস্ব CDN আছে, কিন্তু আলাদা CDN চাইলে Cloudflare ব্যবহার করা যায়।

### SSL সার্টিফিকেট

SSL ছাড়া সাইট লাইভ করা যাবে না — ব্রাউজার "Not Secure" দেখাবে। Vercel অটোমেটিক SSL দেয়, কোনো কনফিগারেশন লাগে না। কাস্টম ডোমেইন যোগ করলেই SSL অটো জেনারেট হয়। লেটস এনক্রিপ্ট ফ্রি SSL প্রোভাইডার, যা ৩ মাসে অটো রিনিউ হয়। Railway তেও SSL ফ্রি, কাস্টম ডোমেইনের সাথে।

### ডোমেইন নির্বাচন

`.com.bd` ডোমেইন বাংলাদেশি ব্যবসার জন্য সেরা। এটি কাস্টমারের ট্রাস্ট বাড়ায় — এটি একটি নিবন্ধিত বাংলাদেশি ব্যবসা। BTCL থেকে নিবন্ধন করতে হয়, ব্যবসার ট্রেড লাইসেন্স লাগে। খরচ বছরে ২,০০০ টাকা। `.com` ডোমেইনও রাখা যায় ইন্টারন্যাশনাল কাস্টমারের জন্য, যদি ভবিষ্যতে এক্সপোর্ট করা হয়।

## ৯. ডেভেলপমেন্ট টুলস

ডেভেলপমেন্ট টুলস টিমের প্রোডাক্টিভিটি বাড়ায় এবং কোড কোয়ালিটি নিশ্চিত করে। এগুলো একবার সেটআপ করলে দীর্ঘমেয়াদে অনেক সময় বাঁচায়। প্রতিটি টুল নির্দিষ্ট সমস্যা সমাধান করে — কোড ফরম্যাটিং, টাইপ চেকিং, টেস্টিং, গিট হুক ইত্যাদি।

| কম্পোনেন্ট | টেকনোলজি | কেন দরকার |
|-----------|---------|----------|
| Monorepo | Turborepo | একাধিক অ্যাপ ম্যানেজ |
| Package Manager | Bun | দ্রুত ইনস্টল |
| TypeScript | TS 5 | টাইপ সেফটি |
| Linting | ESLint + Prettier | কোড কোয়ালিটি |
| Testing | Vitest | ইউনিট টেস্ট |
| Git Hooks | Husky + lint-staged | কমিটের আগে চেক |
| CI/CD | GitHub Actions | অটো টেস্ট + ডিপ্লয় |

### Turborepo

Turborepo হলো জাভাস্ক্রিপ্ট মনোরেপোর জন্য সবচেয়ে দ্রুত বিল্ড সিস্টেম। রিমোট ক্যাশিং — একবার বিল্ড করা কোড আবার বিল্ড করতে হয় না। প্যারালাল এক্সিকিউশন — একাধিক অ্যাপ একসাথে বিল্ড হয়। স্মার্ট রিবিল্ড — শুধু পরিবর্তিত অংশ বিল্ড হয়। Vercel এর তৈরি, তাই Next.js এর সাথে পারফেক্ট ইন্টিগ্রেশন।

### Bun প্যাকেজ ম্যানেজার

Bun হলো সবচেয়ে দ্রুত জাভাস্ক্রিপ্ট রানটাইম ও প্যাকেজ ম্যানেজার। npm এর চেয়ে ১০-২০ গুণ দ্রুত ইনস্টল। নেটিভ TypeScript সাপোর্ট, তাই কম্পাইলেশন দরকার নেই। বিল্ট-ইন টেস্ট রানার, বান্ডলার, মাইগ্রেশন টুল। মেমরি কম ব্যবহার করে। বাংলাদেশের ধীর ইন্টারনেটে এটি বড় সুবিধা।

### TypeScript

TypeScript হলো জাভাস্ক্রিপ্টের সাথে টাইপ সেফটি যোগ করে। কম্পাইল টাইমে এরর ধরা পড়ে, রানটাইমে নয়। IDE অটো-কমপ্লিট ভালো কাজ করে। রিফ্যাক্টরিং নিরাপদ। বড় টিমে কাজ করলে টাইপ সেফটি অপরিহার্য। নতুন ডেভেলপার কোড বুঝতে সহজে পারে। বাগ কম হয়, মেইনটেন্যান্স সহজ।

### ESLint + Prettier

ESLint কোড কোয়ালিটি চেক করে — ব্যবহৃত না হওয়া ভেরিয়েবল, সম্ভাব্য বাগ, বেস্ট প্র্যাকটিস ভায়োলেশন। Prettier কোড ফরম্যাট করে — ইন্ডেন্টেশন, কোটেশন, স্পেসিং। দুটো একসাথে কোড কনসিস্টেন্ট রাখে। টিমের সবার কোড একই স্টাইলে থাকে। কোড রিভিউতে স্টাইল নিয়ে ডিসকাশন কম হয়।

### Vitest টেস্টিং

Vitest হলো Vite-পাওয়ার্ড টেস্ট রানার, Jest এর মতো কিন্তু দ্রুত। ইউনিট টেস্ট, ইন্টিগ্রেশন টেস্ট — সব সাপোর্ট করে। ওয়াচ মোড দ্রুত রিরান করে। TypeScript নেটিভ সাপোর্ট। কভারেজ রিপোর্ট জেনারেট করে। টেস্ট না থাকলে রিফ্যাক্টরিং ভয়ের কারণ হয়ে দাঁড়ায়।

### Husky + lint-staged

Husky গিট হুক ম্যানেজ করে। lint-staged শুধু পরিবর্তিত ফাইলে lint চালায়। কমিটের আগে অটোমেটিক লিন্ট ও ফরম্যাট হয়। খারাপ কোড মেইন ব্রাঞ্চে যেতে পারে না। টিমের সবাই একই স্ট্যান্ডার্ড ফলো করে।

### GitHub Actions CI/CD

GitHub Actions দিয়ে অটোমেশন — পুশ করলে টেস্ট চলে, বিল্ড হয়, ডিপ্লয় হয়। PR খুললে অটো চেক। মেইন ব্রাঞ্চে মার্জ হলে প্রোডাকশনে ডিপ্লয়। ম্যানুয়াল ডিপ্লয় লাগে না। ফ্রি টিয়ারে মাসে ২০০০ মিনিট, যা ছোট প্রজেক্টের জন্য যথেষ্ট।

## ১০. AI ও ইন্টেলিজেন্স

AI ব্যবহার করে ePowerFix একজন ভার্চুয়াল এমপ্লয়ি পেতে পারে যে ২৪/৭ কাজ করে। কাস্টমার সাপোর্ট, কন্টেন্ট জেনারেশন, মার্কেটিং অটোমেশন — সব ক্ষেত্রে AI সময় ও খরচ বাঁচায়। z-ai-web-dev-sdk ব্যবহার করে সব AI ফিচার ইন্টিগ্রেট করা যায়।

| কম্পোনেন্ট | টেকনোলজি | কেন দরকার |
|-----------|---------|----------|
| AI Chatbot | z-ai-web-dev-sdk (LLM) | কাস্টমার সাপোর্ট |
| Image Generation | z-ai-web-dev-sdk | মার্কেটিং ইমেজ |
| TTS | z-ai-web-dev-sdk | ভিডিও ভয়েসওভার |
| Web Search | z-ai-web-dev-sdk | প্রতিযোগী মনিটরিং |

### AI চ্যাটবট

AI চ্যাটবট ২৪/৭ কাস্টমার সাপোর্ট দেয়। প্রোডাক্ট ইনফো, দাম, স্টক, ডেলিভারি চার্জ — সব প্রশ্নের উত্তর দেয়। সার্ভিস বুকিং হেল্প করে। সাধারণ ইলেকট্রিক্যাল প্রশ্নের উত্তর দেয়। এর ফলে ১ জন সাপোর্ট স্টাফের কাজ বেঁচে যায়। চ্যাট হিস্ট্রি সেভ থাকে, তাই কনটেক্সট বুঝতে পারে। কমপ্লেক্স প্রশ্ন হস্তান্তর করে হিউম্যান এজেন্টকে।

### ইমেজ জেনারেশন

AI দিয়ে মার্কেটিং ইমেজ তৈরি — সোশ্যাল মিডিয়া পোস্টার, প্রোডাক্ট থাম্বনেইল, ব্যানার ডিজাইন। ডিজাইনার না ভাড়া করেই প্রফেশনাল ইমেজ পাওয়া যায়। প্রম্পট দিয়ে কাস্টমাইজ করা যায়। বাল্ক জেনারেশন সাপোর্ট। প্রতিটি প্রোডাক্টের জন্য আলাদা ইমেজ।

### TTS টেক্সট-টু-স্পিচ

TTS দিয়ে ইউটিউব ভিডিওর ভয়েসওভার তৈরি করা যায়। নিজে কথা না বলেও প্রফেশনাল ভয়েস পাওয়া যায়। বাংলা ভয়েস সাপোর্ট করে। স্ক্রিপ্ট লিখে দিলেই অডিও রেডি। ভিডিও প্রোডাকশন খরচ অনেক কমে যায়।

### ওয়েব সার্চ

প্রতিযোগী মনিটরিং — Daraz, অনলাবা, লোকাল শপের দাম ট্র্যাক করা। নতুন ট্রেন্ড খুঁজে বের করা — IoT, Solar, Smart Home। সাপ্লায়ার রিসার্চ। মার্কেট রিসার্চ অটোমেটেড। ম্যানুয়ালি সার্চ করার সময় বাঁচে।

## ১১. সম্পূর্ণ স্ট্রাকচার

এখানে সম্পূর্ণ মনোরেপো স্ট্রাকচার দেওয়া হলো। প্রতিটি ফোল্ডারের নির্দিষ্ট দায়িত্ব আছে। apps/ ফোল্ডারে রানেবল অ্যাপ, packages/ ফোল্ডারে শেয়ার্ড লাইব্রেরি, prisma/ ফোল্ডারে ডেটাবেস।

```
ePowerFix/
├── apps/
│   ├── web/                    ← Next.js (UI only)
│   │   ├── src/
│   │   │   ├── app/            ← পেজ
│   │   │   └── components/     ← ওয়েব কম্পোনেন্ট
│   │   └── package.json
│   │
│   ├── mobile/                 ← Expo (mobile app)
│   │   ├── app/                ← স্ক্রিন
│   │   └── package.json
│   │
│   └── api/                    ← Express (backend)
│       ├── src/
│       │   ├── routes/         ← API রুট
│       │   ├── controllers/    ← বিজনেস লজিক
│       │   ├── middleware/     ← auth, rateLimit, validate
│       │   ├── services/       ← ডেটাবেস অপারেশন
│       │   ├── utils/          ← হেল্পার

│       └── package.json
│
├── packages/                   ← শেয়ার্ড কোড
│   ├── types/                  ← TypeScript টাইপ
│   ├── api-client/             ← fetch wrapper
│   ├── store/                  ← Zustand স্টোর
│   ├── utils/                  ← ইউটিলিটি
│   ├── config/                 ← কনফিগ, এনভায়রনমেন্ট
│   └── ui/                     ← শেয়ার্ড UI (NativeWind)

├── prisma/                     ← ডেটাবেস স্কিমা (shared)

├── turbo.json
└── package.json
```

### ফোল্ডার ব্যাখ্যা

`apps/web/` — Next.js ওয়েব অ্যাপ। শুধু UI কম্পোনেন্ট ও পেজ থাকবে। কোনো API রুট এখানে থাকবে না। Vercel-এ ডিপ্লয় হবে। `apps/mobile/` — Expo মোবাইল অ্যাপ। React Native স্ক্রিন থাকবে। EAS দিয়ে বিল্ড হবে। Play Store ও App Store-এ সাবমিট হবে। `apps/api/` — Express ব্যাকএন্ড। সব API রুট, বিজনেস লজিক, ডেটাবেস অপারেশন এখানে। Railway-এ ডিপ্লয় হবে।

`packages/` — শেয়ার্ড কোড। তিনটি অ্যাপই এগুলো ব্যবহার করবে। টাইপ, API ক্লায়েন্ট, স্টোর, ইউটিলিটি — সব এক জায়গায়। পরিবর্তন করলে সব অ্যাপে রিফ্লেক্ট হয়। `prisma/` — ডেটাবেস স্কিমা ও মাইগ্রেশন। সব অ্যাপ এই স্কিমা ব্যবহার করে।

## সারসংক্ষেপ

ePowerFix প্ল্যাটফর্মের জন্য সম্পূর্ণ স্ট্যাক প্রস্তুত। মূল চারটি কম্পোনেন্ট (Turborepo, Next.js, Express, PostgreSQL) ছাড়াও ২৩টি অতিরিক্ত কম্পোনেন্ট যোগ করা হয়েছে। প্রতিটি কম্পোনেন্ট নির্দিষ্ট সমস্যা সমাধান করে এবং প্রোডাকশন-রেডি সিস্টেমের জন্য অপরিহার্য।

**মোট ২৭টি কম্পোনেন্ট** নিয়ে সম্পূর্ণ স্ট্যাক তৈরি। শুরুতে সব ফ্রি টিয়ারে চলবে, মাত্র ২,০০০ ৳/বছর খরচ (ডোমেইন)। ব্যবসা বাড়লে পেইড প্ল্যানে যাওয়া যাবে, মাসিক খরচ $১৫০-২৫০।

এই আর্কিটেকচার বাংলাদেশের প্রেক্ষাপটে সেরা — বাংলাদেশি পেমেন্ট গেটওয়ে, লোকাল ডেলিভারি, বাংলা ভাষা সাপোর্ট, এবং কম খরচে স্কেল।
